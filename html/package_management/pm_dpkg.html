dpkg

>Usage

The dpkg command is a low-level package manager for Debian-based systems, including Ubuntu. It is primarily used to install, manage, and remove .deb packages (Debian software packages). Unlike apt, which manages dependencies and can fetch packages from online repositories, dpkg works directly with local .deb files and does not resolve dependencies automatically.

>Synopsis

dpkg [option] [package]

>Options

-c

Lists the contents of a .deb package file without installing it. This is helpful to see what files will be installed.

Example:

-c

> dpkg -c example-package.deb

drwxr-xr-x root/root 0 2024-11-14 09:45 ./usr/
drwxr-xr-x root/root 0 2024-11-14 09:45 ./usr/bin/
-rwxr-xr-x root/root 123456 2024-11-14 09:45 ./usr/bin/example
drwxr-xr-x root/root 0 2024-11-14 09:45 ./usr/share/doc/example/
-rw-r--r-- root/root 12345 2024-11-14 09:45 ./usr/share/doc/example/README.md

-L

Lists all the files installed on the system by a specific package. This is useful for locating where a package installed its files.

Example:

-L

> dpkg -L example-package

/. /usr
/usr/bin
/usr/bin/example
/usr/share
/usr/share/doc
/usr/share/doc/example
/usr/share/doc/example/README.md

-s

Shows the status and details of an installed package, including its version, description, and dependencies.

Example:

-s

> dpkg -s example-package

Package: example-package
Status: install ok installed
Priority: optional
Section: utils
Installed-Size: 256
Maintainer: Example Maintainer
Architecture: amd64
Version: 1.2.3-1
Description: This is an example package for demonstration purposes.
